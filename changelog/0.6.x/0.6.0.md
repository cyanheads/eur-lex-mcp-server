# 0.6.0 — 2026-07-03

## Added

- **`eurlex_get_relations` repeal relations** ([#31](https://github.com/cyanheads/eur-lex-mcp-server/issues/31)) — four new `relation_types` values: `repeals`/`repealed_by` (outgoing/incoming sides of `cdm:resource_legal_repeals_resource_legal`) and `implicitly_repeals`/`implicitly_repealed_by` (outgoing/incoming sides of `cdm:resource_legal_implicitly_repeals_resource_legal`), the same predicate-inversion pattern `amends`/`amended_by` already uses. Inherited by the `eurlex://document/{celex}/relations` resource.

## Changed

- **`consolidated_version` filtering** ([#32](https://github.com/cyanheads/eur-lex-mcp-server/issues/32)) — rows with no related CELEX number are now always dropped (unfetchable via `eurlex_get_document`), and when the source CELEX is known, rows whose CELEX act core (`{year}{type}{number}`) doesn't match the source's are dropped too, filtering out CELLAR graph artifacts such as a consolidation of the act this one repealed. Shared by `eurlex_get_relations` and the `eurlex://document/{celex}/relations` resource.
