# 0.9.3 — 2026-07-04

## Added

- **`eurlex_get_cases` `include_derivative`** (default `false`) ([#44](https://github.com/cyanheads/eur-lex-mcp-server/issues/44)) — re-admits derivative sector-6 case-law records (judicial information notices, case abstracts, case summaries) into search results. Ignored when `case_type` is set, since that path already returns a single primary type.
- **`eurlex_get_relations` `requested_relation_types` / `empty_relation_types`** ([#47](https://github.com/cyanheads/eur-lex-mcp-server/issues/47)) — additive output fields: the effective requested relation types, and the subset with zero relations in the current page. Rendered in `format()` for non-`structuredContent` clients.

## Changed

- **`cdm-labels.ts`** ([#44](https://github.com/cyanheads/eur-lex-mcp-server/issues/44)) — `INFO_JUDICIAL`, `INFO_JUR`, `ABSTRACT_JUR`, and `SUM_JUR` resource-types now resolve to labels ("Judicial Information Notice", "Information Notice", "Case Abstract", "Case Summary") instead of raw CDM codes, for records re-admitted via `include_derivative`.

### Dependency bumps

- `tsc-alias` (dev) `^1.8.17` → `^1.9.0`

## Fixed

- **`eurlex_get_cases` default search crowded out by derivative records** ([#44](https://github.com/cyanheads/eur-lex-mcp-server/issues/44)) — the untyped/default search path now excludes derivative sector-6 records via a SPARQL `FILTER NOT EXISTS`, so distinct primary cases fill the page instead of being displaced by notices, abstracts, and summaries of the same case. Type-less older records carry none of the derivative types and are unaffected.
- **`eurlex_get_relations` false `consolidated_version` truncation** ([#45](https://github.com/cyanheads/eur-lex-mcp-server/issues/45)) — the CELEX validity filter (requiring the related CELEX, and — when the source act core is known — a `REGEX` pinning it to a genuine same-act consolidation) is now pushed into the SPARQL query instead of applied only client-side; the per-direction truncation counter counts post-filter rows, so a page filled by filtered-out graph artifacts no longer reports `truncated: true`. Shared by the tool and the `eurlex://document/{celex}/relations` resource via `traverseRelations`.
