# 0.9.7 — 2026-07-16

## Added

- **`query_echo` echoes effective result-shaping defaults** ([#57](https://github.com/cyanheads/eur-lex-mcp-server/issues/57)) — `eurlex_search_documents.query_echo` now carries `include_consolidated` and `eurlex_get_cases.query_echo` carries `include_derivative`, each populated with the effective value after the `false` default is applied and rendered in `content[]` through the existing filter summary, so `structuredContent` and the text channel both describe the full search semantics. A defaulted `false` still appears — it shapes which records can return — while both defaults and the underlying consolidated/derivative filtering are unchanged.

## Changed

- **`eurlex_get_cases` excludes standalone corrigenda by default** ([#55](https://github.com/cyanheads/eur-lex-mcp-server/issues/55)) — `CORRIGENDUM` joins the derivative sector-6 resource-type exclusion (`INFO_JUDICIAL`, `INFO_JUR`, `ABSTRACT_JUR`, `SUM_JUR`), so a work typed only `CORRIGENDUM` cannot take a primary-case slot on the default path; `include_derivative: true` re-admits it. Makes the exclusion explicit rather than relying on CELLAR co-tagging corrigenda with an already-excluded type — no primary judgment, order, or AG opinion is typed `CORRIGENDUM`, so real-case recall is unaffected.
- **`docs/design.md` and `README.md` reconciled to the live content path** ([#59](https://github.com/cyanheads/eur-lex-mcp-server/issues/59)) — the design doc and the README project-structure row now describe `EurLexContentService`'s current flow (CELLAR content negotiation at `publications.europa.eu/resource/celex/{CELEX}` with `Accept`/`Accept-Language`, XHTML→HTML fallback, multipart Formex assembly, English fallback, and AWS-WAF challenge detection) and the `bif:contains` full-text keyword search, replacing the stale `eur-lex.europa.eu/legal-content` REST-path description. Docs only; no runtime change.
