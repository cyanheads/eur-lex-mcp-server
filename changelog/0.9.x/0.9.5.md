# 0.9.5 — 2026-07-09

## Added

- **`eurlex_browse_subjects` `offset`** ([#51](https://github.com/cyanheads/eur-lex-mcp-server/issues/51)) — pagination offset input (default `0`) plus a matching `offset` output field, for paging EuroVoc concepts past the first page.

## Changed

- **`eurlex_browse_subjects` distinct, alphabetically-ordered concepts** ([#51](https://github.com/cyanheads/eur-lex-mcp-server/issues/51)) — the SPARQL now `GROUP BY`s concept and label with a deterministic `ORDER BY ?label ?concept` and `LIMIT`/`OFFSET`, collapsing the per-(notation × broader-parent) row explosion that let a single polyhierarchical concept consume several page slots. Tool and output descriptions drop the inaccurate "ordered by relevance" wording for "ordered alphabetically by label."

## Fixed

- **`eurlex_query_sparql` text output truncated to 20 rows** ([#50](https://github.com/cyanheads/eur-lex-mcp-server/issues/50)) — `format()` rendered only the first 20 bindings while `structuredContent` carried the full set (bounded only by the service-side `maxSparqlResults` cap), leaving `content[]`-only clients blind to rows 21+. The text table now renders every returned row.
