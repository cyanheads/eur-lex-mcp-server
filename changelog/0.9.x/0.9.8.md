# 0.9.8 — 2026-07-16

## Fixed

- **`eurlex_query_sparql` result cap now bounds the outermost query, not the first `LIMIT` in the text** ([#63](https://github.com/cyanheads/eur-lex-mcp-server/issues/63)) — `enforceLimitInQuery` enforced the `maxSparqlResults` ceiling on whichever `LIMIT` appeared first, so a subselect's `LIMIT` absorbed the rewrite: the outer query was left uncapped and the caller's own inner `LIMIT` was silently rewritten down to the ceiling. Enforcement now targets the outermost (brace-depth 0) `LIMIT` and is scoped to the raw escape hatch (`queryWithVars`) — a subselect-only `LIMIT` gets an outer bound appended with the inner one left intact, and a top-level `LIMIT` above the ceiling is rewritten in place, preserving `ORDER BY`/`GROUP BY`. Internal self-bounded queries (the symmetric relation-traversal UNION) pass through unchanged.
