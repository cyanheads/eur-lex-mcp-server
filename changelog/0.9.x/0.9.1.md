# 0.9.1 — 2026-07-03

## Changed

- **Tool and resource descriptions** ([#36](https://github.com/cyanheads/eur-lex-mcp-server/issues/36)) — `eurlex_browse_subjects`, `eurlex_search_documents`, `eurlex_get_relations`, `eurlex_get_document`, `eurlex_get_cases`, `eurlex_lookup_celex`, `eurlex_query_sparql`, and both document resources now lead with capability and output instead of mixing in cross-tool workflow coaching ("use X before Y"); domain constraints (CELEX/ELI formats, `eurlex_query_sparql` SELECT-only) are unchanged.
- **`createApp({ instructions })`** ([#36](https://github.com/cyanheads/eur-lex-mcp-server/issues/36)) — gains a workflow-orientation section (subject search, legislation vs. case law, full text, relationship traversal, identifier validation) and an identifiers section (CELEX/ELI formats), carrying the cross-tool guidance trimmed from individual descriptions.

## Fixed

- **`eurlex_comparative_analysis` duplicate focus sections** ([#37](https://github.com/cyanheads/eur-lex-mcp-server/issues/37)) — an optional `focus` overlapping an existing analysis axis (e.g. "enforcement mechanisms") now retitles that axis instead of appending a duplicate "Deep dive" section; a non-overlapping focus still gets its own dedicated section. Built on a fixed `ANALYSIS_AXES` array and a `matchFocusToAxis()` keyword mapper.
