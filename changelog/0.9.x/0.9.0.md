# 0.9.0 — 2026-07-03

## Added

- **`eurlex_get_relations` `offset` + `limit` inputs** ([#39](https://github.com/cyanheads/eur-lex-mcp-server/issues/39)) — `limit` (1–100, default 100) sets the per-relation-type, per-direction cap; `offset` pages past it. Output gains a matching `offset` field.
- **`eurlex_get_relations` truncation disclosure** ([#39](https://github.com/cyanheads/eur-lex-mcp-server/issues/39)) — `ctx.enrich.truncated({shown, cap})` fires when a direction fills its cap, the same signal `eurlex_get_cases` and `eurlex_search_documents` expose; also surfaced as `truncated` on the `eurlex://document/{celexNumber}/relations` resource.
- **`author_institutions` output** on `eurlex://document/{celexNumber}` ([#35](https://github.com/cyanheads/eur-lex-mcp-server/issues/35)) — the full de-duplicated set of co-legislating author institution labels (e.g. Parliament + Council for GDPR); `author_institution` remains the primary (first).

## Fixed

- **`eurlex_get_relations` incoming citations silently dropped the newest** ([#39](https://github.com/cyanheads/eur-lex-mcp-server/issues/39)) — each relation query now runs `ORDER BY DESC(?relatedDate)`; the symmetric `cites` relation splits into independently-capped, independently-ordered UNION subqueries per direction so a dense outgoing set no longer crowds out incoming.
- **`eurlex_get_document` `format()` truncated EuroVoc subjects to 5** ([#41](https://github.com/cyanheads/eur-lex-mcp-server/issues/41)) — the text channel now renders the full list, matching `structuredContent`.
- **`resource_type` / `author_institution` returned raw CDM authority URIs** on `eurlex://document/{celexNumber}` ([#35](https://github.com/cyanheads/eur-lex-mcp-server/issues/35)) — now resolved to human-readable labels (e.g. `Regulation`, `Council of the EU`) via the same helpers `eurlex_get_document` uses. Breaking: code that parsed these fields as authority URIs must adapt to the label strings.
- **`cdm-labels` implementing/delegated regulation keys transposed** ([#43](https://github.com/cyanheads/eur-lex-mcp-server/issues/43)) — `RESOURCE_TYPE_LABELS` keyed them `IMPL_REG`/`DEL_REG`; CELLAR's authority register emits `REG_IMPL`/`REG_DEL`. Both types now resolve correctly across `eurlex_get_document`, `eurlex_search_documents`, and the document resource.
