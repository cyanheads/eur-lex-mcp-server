# 0.1.7 — 2026-06-30

## Fixed

- **`eurlex_search_documents` author_institution filter** ([#6](https://github.com/cyanheads/eur-lex-mcp-server/issues/6)) — the author match sat inside an `OPTIONAL` block over `cdm:corporate-body_label` (a predicate CELLAR does not expose), so it never constrained selection and an impossible author still returned normal rows. It is now a required graph pattern matching the institution's English `skos:prefLabel` through Virtuoso's `bif:contains` full-text index; the value is reduced to letters, digits, and spaces before embedding, and an author that matches nothing — or sanitizes to empty — returns `no_results`.
- **`eurlex_get_document` and `eurlex://document/{celex}` titles** ([#7](https://github.com/cyanheads/eur-lex-mcp-server/issues/7)) — both metadata queries used the obsolete `?work cdm:work_title` traversal, which returns nothing in current CELLAR, so titles were omitted. They now use the expression-level traversal (`expression_belongs_to_work` + English `expression_uses_language` + `expression_title`) already adopted for search and cases.

## Changed

- **`eurlex_query_sparql` description** — corrected the stale note that Virtuoso cannot `bif:contains` multi-word phrases; single-quoting the phrase (e.g. `"'data protection'"`) uses the full-text index, with `FILTER(CONTAINS(LCASE(?title), "keyword"))` as the scan fallback.
