# 0.1.3 — 2026-06-06

## Added

- **`cdm-labels.ts`** — shared CDM authority-register URI→label module (`ENG_LANGUAGE_URI`, `RESOURCE_TYPE_LABELS`, `CORPORATE_BODY_LABELS`, `resolveResourceTypeLabel()`, `resolveCorporateBodyLabel()`); consolidated from per-tool inline maps ([#4](https://github.com/cyanheads/eur-lex-mcp-server/issues/4)).
- **`query_echo`** — both `eurlex_search_documents` and `eurlex_get_cases` now return applied filters in `structuredContent`; empty-result error messages embed the same summary ([#3](https://github.com/cyanheads/eur-lex-mcp-server/issues/3)).

## Fixed

- **`eurlex_search_documents` / `eurlex_get_cases` keyword search** — replaced `cdm:work_title` traversal (which has no triples at the work level) with the correct expression-level path: `?expr cdm:expression_belongs_to_work ?work` + `cdm:expression_uses_language <.../ENG>` + `cdm:expression_title ?title` ([#1](https://github.com/cyanheads/eur-lex-mcp-server/issues/1)).
- **`eurlex_get_cases` case number resolution** — `C-131/12` now converts to the correct CELEX substring `2012CJ0131` (year-first, zero-padded); the old approach searched the raw `131/12` string, which inverts the CELEX format and never matched ([#2](https://github.com/cyanheads/eur-lex-mcp-server/issues/2)).
- **`resource_type` / `author_institution` labels** — `eurlex_get_document`, `eurlex_search_documents`, and `eurlex_get_cases` resolve CDM authority URIs to human-readable labels (e.g. `"Regulation"`, `"Council of the EU"`) instead of returning raw URIs ([#4](https://github.com/cyanheads/eur-lex-mcp-server/issues/4)).
