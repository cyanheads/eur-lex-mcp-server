# 0.1.4 — 2026-06-11

## Added

- **Server identity pair** — `createApp()` now sets explicit `name: 'eur-lex-mcp-server'` and `title: 'eur-lex-mcp-server'`, so the machine name shows on every client surface and `lint:packaging` check 9 passes.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script: runs `mcpb clean` (dev-dependency prune) then strips `node_modules/**` agent-doc entries (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **Dockerfile `HEALTHCHECK`** — `bun -e fetch(...)` against `/healthz` every 30s; no `curl`/`wget` dependency in the slim runtime image.
- **Dockerfile `APP_VERSION` build arg** — stamps `org.opencontainers.image.version` on built images; the release `docker buildx` passes `--build-arg APP_VERSION`.

## Changed

- **SPARQL error classification** — malformed-query and ambiguous-identifier failures now throw `ValidationError` instead of `InvalidParams` across `eurlex_query_sparql`, `eurlex_lookup_celex`, and the CELLAR SPARQL service (`validationError()` replaces `invalidParams()`). `ValidationError` is the framework baseline code for bad input.
- **`bundle` script** — now Bun-native (`bun run build`) and appends the `clean-mcpb.ts` step after `mcpb pack`.
- **`.mcpbignore`** — dev-dir patterns re-anchored to root (`/.claude/`, `/.agents/`, `/skills/`, `/Dockerfile`, etc.) so they can't strip nested `node_modules` runtime paths.
- **Framework tooling sync** — `check-framework-antipatterns.ts` gains the `coerce-boolean-env-flag` rule (and skips comment lines); `lint-packaging.ts` gains post-bundle content (check 8) and entrypoint/manifest identity (check 9) checks.
- **Project skills re-synced** to the framework 0.10.x baseline; added the `techniques` skill (outline-on-overflow catalog).
