# 0.5.0 — 2026-07-01

## Added

- **`eurlex_get_document` outline mode and section selectors** ([#12](https://github.com/cyanheads/eur-lex-mcp-server/issues/12)) — `outline: true` returns the act's chapters, sections, articles, annexes, and recitals as a heading list with character offsets and no body; `select` (e.g. `{ articles: "1,5,17" }`, Roman or Arabic numbering) returns only those sections' text. New output fields `outline`, `selection` (`requested`/`matched`/`missed`), and `structure_detected`. Detection is best-effort across `html`/`markdown` (rendered-text patterns) and `xml` (Formex elements); an act with no parseable structure (e.g. case law) degrades to the paging floor with a notice, never an error. Adds `src/services/eurlex-content/act-structure.ts`.
- **Multi-part Formex 4 assembly for `format: "xml"`** ([#18](https://github.com/cyanheads/eur-lex-mcp-server/issues/18)) — acts whose Formex 4 is served across multiple streams (CELLAR answers with HTTP 300 Multiple Choices, e.g. the GDPR `32016R0679`) returned `content_available: false` on the XML path. The service now follows the sibling part references and concatenates them in `DOC_<n>` order into one wrapped document. Best-effort: if any part cannot be fetched it falls back to the existing `content_available: false` rather than throwing. The `html`/`markdown` paths are unchanged.

## Changed

- **Dependencies** — `@biomejs/biome` ^2.5.1 → ^2.5.2, `@types/node` ^26.0.1 → ^26.1.0.
