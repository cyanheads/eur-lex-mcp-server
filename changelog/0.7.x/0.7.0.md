# 0.7.0 — 2026-07-03

## Added

- **`work_uri` input** on `eurlex_get_document` ([#34](https://github.com/cyanheads/eur-lex-mcp-server/issues/34)) — accepts a CELLAR work resource URI (the form returned by `eurlex_lookup_celex`, `eurlex_get_relations`, and `eurlex_search_documents`), dereferenced via `cdm:resource_legal_id_celex` to its CELEX and fetched by the existing flow. Exactly one of `celex_number`, `eli_uri`, or `work_uri` is required.
- **`author_institutions` output** on `eurlex_get_document` ([#33](https://github.com/cyanheads/eur-lex-mcp-server/issues/33)) — the full set of co-legislating author institutions (e.g. `["European Parliament", "Council of the EU"]` for an ordinary-legislative-procedure act); `author_institution` remains the primary (first) for back-compat.
- **Staleness signal + `resolve` input** on `eurlex_get_document` ([#29](https://github.com/cyanheads/eur-lex-mcp-server/issues/29)) — `is_superseded`, `current_consolidated_celex`, and `consolidated_as_of` flag a base act with a newer consolidated version; the opt-in `resolve: "current_consolidated"` input transparently serves that version instead and echoes the original request in `requested_celex`.

## Changed

- **`eurlex_get_document` metadata fetch** ([#33](https://github.com/cyanheads/eur-lex-mcp-server/issues/33)) — restructured from a single `LIMIT 20` cross-product query over the author/legal-basis/EuroVoc `OPTIONAL`s into a core query plus one concurrent `LIMIT 100` query per multi-valued dimension, so no dimension can truncate another.

## Fixed

- **`author_institution` dropped co-legislators** ([#33](https://github.com/cyanheads/eur-lex-mcp-server/issues/33)) — `eurlex_get_document` read the author from the first metadata row only, so acts adopted by more than one institution (e.g. GDPR: European Parliament + Council of the EU) reported just one.
