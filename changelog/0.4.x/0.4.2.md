# 0.4.2 — 2026-06-30

## Changed

- **Keyword search uses the full-text index** ([#17](https://github.com/cyanheads/eur-lex-mcp-server/issues/17)) — `eurlex_search_documents` and `eurlex_get_cases` matched the keyword against titles with a `FILTER(CONTAINS(LCASE(?title), …))` scan, forcing Virtuoso to join the expression graph and test every candidate title across the 2.7M-work corpus, so a broad or lightly-filtered keyword risked the query timeout. Title matching now runs through `bif:contains` on the Virtuoso full-text index — the mechanism the `author_institution` filter already uses. Multi-word input is matched as a title phrase rather than a substring; exact-substring CELEX matching is preserved as a UNION arm.

## Fixed

- **`eurlex_get_cases` duplicate rows for one CELEX** ([#21](https://github.com/cyanheads/eur-lex-mcp-server/issues/21)) — CELLAR exposes several distinct work URIs under one CELEX (a titled judgment plus a `do_not_index` member work), and grouping by `?work` returned each as its own row — one with the title, one bare — so a page of N surfaced fewer than N distinct cases. The query now groups by CELEX, collapsing the duplicates into one row per case that keeps the titled work's URI and title; `total` counts distinct cases.
- **`eurlex_search_documents` duplicate rows for one CELEX** ([#24](https://github.com/cyanheads/eur-lex-mcp-server/issues/24)) — same cause and fix: distinct works sharing a CELEX (a titled work plus a `do_not_index` member, or parallel manifestations) each consumed a result slot, so a `limit: N` page could surface fewer than N distinct documents. Grouping by CELEX collapses them into one row per document.
