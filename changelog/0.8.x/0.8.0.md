# 0.8.0 — 2026-07-03

## Added

- **`include_consolidated` input + `is_consolidated` output** on `eurlex_search_documents` ([#30](https://github.com/cyanheads/eur-lex-mcp-server/issues/30)) — consolidated texts (CONS_TEXT) carry a resource-type distinct from their base act's `document_type`, so a type filter silently excluded them; `include_consolidated: true` widens the SPARQL filter to also admit CONS_TEXT, and every result row is tagged `is_consolidated`.

## Fixed

- **Whitespace-only `keyword` no longer scans the full corpus** ([#25](https://github.com/cyanheads/eur-lex-mcp-server/issues/25)) — `eurlex_search_documents` now rejects a search with no effective narrowing filter via a new `no_filters` error instead of issuing an unbounded query against the 2.7M-work corpus; both `eurlex_search_documents` and `eurlex_get_cases` trim the keyword and omit it from `query_echo` when blank, instead of echoing raw whitespace as if it had filtered.
