# 0.8.1 — 2026-07-03

## Added

- **`truncated`/`shown`/`cap` enrichment** on `eurlex_search_documents`, `eurlex_get_cases`, and `eurlex_browse_subjects` ([#28](https://github.com/cyanheads/eur-lex-mcp-server/issues/28)) — a full page now discloses that more results may exist, via `ctx.enrich.truncated(...)`, so callers can distinguish a capped page from the complete result set.

## Changed

- **Reader-/config-specific phrasing removed** from `eurlex_browse_subjects`'s description and `eurlex_query_sparql`'s `timeout_hint` description ([#27](https://github.com/cyanheads/eur-lex-mcp-server/issues/27)) — no more "agents cannot guess…" framing or a named `SPARQL_QUERY_TIMEOUT_MS` env var in caller-facing text.

## Fixed

- **`sparql_error` responses carry their declared recovery hint** ([#26](https://github.com/cyanheads/eur-lex-mcp-server/issues/26)) — `cellar-sparql-service.ts` now attaches `recovery: { hint: SPARQL_ERROR_RECOVERY_HINT }` to both `validationError` throws, so `data.recovery.hint` and the mirrored `Recovery:` content line are populated for `eurlex_query_sparql` and every other tool that queries through the shared CELLAR SPARQL service.
